PROMPT: Describe the binary chop algorithm in depth
RESPONSE: The **binary chop algorithm**, more commonly known as **binary search**, is a highly efficient algorithm for finding a target value within a **sorted** array (or list). It works by repeatedly dividing the search interval in half, eliminating half of the remaining elements in each step. Its time complexity is **O(log n)**—significantly faster than linear search’s O(n) for large datasets.

Below is a comprehensive, in-depth explanation:

---

### 🔑 Core Principles

1. **Requirement: Sorted Data**  
   The array must be sorted (e.g., ascending or descending). Binary search *cannot* be applied directly to unsorted data (unless sorting is done first).

2. **Divide-and-Conquer Strategy**  
   At each step, the algorithm compares the target value to the middle element of the current search interval:
   - If equal → found.
   - If target < middle → search the *left half*.
   - If target > middle → search the *right half*.

3. **Halving the Search Space**  
   Each comparison reduces the problem size by ~50%, leading to logarithmic growth in steps.

RESPONSE: Here's a clean, idiomatic quicksort implementation in Rust that uses in-place partitioning and handles edge cases safely:

```rust
fn quicksort<T: Ord + Clone>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    // Partition the array around a pivot (using the last element)
    let pivot_index = partition(arr);
    
    // Recursively sort the two partitions
    quicksort(&mut arr[..pivot_index);
    quicksort(&mut arr[pivot_index + 1..]);
}

// Helper function to partition the array
fn partition<T: Ord + Clone>(arr: &mut [T>) -> usize {
    let pivot = arr.last().unwrap().clone();
    let mut i = 0;
    
    for j in 0..arr.len() - 1 {
        if arr[j] <= pivot {
            arr.swap(i, j);
            i += 1;
        }
    }
    
    arr.swap(i, arr.len() - 1);
    i
}
```
