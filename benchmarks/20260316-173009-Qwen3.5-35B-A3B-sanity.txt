PROMPT: Write me a quicksort implementation in Rust
RESPONSE: 

Here is an idiomatic, in-place Quicksort implementation in Rust.

It uses a mutable slice (`&mut [T]`), generic types constrained by `Ord`, and recursion. It avoids unnecessary allocations by sorting the array in place.

```rust
/// Sorts a slice in place using the Quicksort algorithm.
/// 
/// # Type Parameters
/// * `T`: The type of the elements, which must implement `Ord` for comparison.
/// 
/// # Panics
/// This function will panic if `i` or `j` are out of bounds during the swap,
/// but the logic below guarantees indices remain within valid bounds.
fn quicksort<T: Ord>(arr: &mut [T]) {
    let len = arr.len();
    
    // Base case: arrays of size 0 or 1 are already sorted
    if len <= 1 {
        return;
    }

    // 1. Choose a pivot. Here we pick the middle element.
    let pivot_idx = len / 2;
    
    // Move the pivot to the end of the slice for easier partitioning
    // (We swap it to index 0 first, then to the end)
    arr.swap(0, pivot_idx);
    arr.swap(0, len - 1);

    // 2. Partitioning (Lomuto scheme)
    // `split` tracks the boundary between elements < pivot and elements >= pivot
    let mut split = 0;
    
    // Iterate through the slice excluding the last element (the pivot)
    for i in 0..(len - 1) {
        // If current element is smaller than pivot, move it to the left side
        if arr[i] < arr[len - 1] {
            arr.swap(i, split);
            split += 1;
        }
    }
    
    // Place the pivot in its correct sorted position
    arr.swap(len - 1, split);

    // 3. Recursive calls
    // Sort the left part (elements smaller than pivot)
    if split > 0 {
        quicksort(&mut arr[..split]);
    }
    
    // Sort the right part (elements larger than pivot)
    if split + 1 < len {
        quicksort(&mut arr[split + 1..]);
    }
}

fn main() {
    let mut data = vec![3, 6, 8, 10, 1, 2, 1];
    
    println!("Before: {:?}", data);
    quicksort(&mut data);
    println!("After:  {:?}", data);
    
    // Test with strings (which also implement Ord)
    let mut words = vec!["pear", "apple", "banana"];
    println!("Words Before: {:?}", words);
    quicksort(&mut words);
    println!("Words After:  {:?}", words);
}
```

### Key Implementation Details

1.  **Generics (`<T: Ord>`)**: The function works on any type that implements the `Ord` trait (like `i32`, `String`, `char`), allowing for generic comparisons without needing `Clone` or `Copy`.
2.  **In-Place**: It takes `&mut [T]` (a mutable slice) and sorts the data directly in memory, avoiding the allocation overhead of creating new vectors during recursion.
3.  **Pivot Strategy**: It picks the middle element as a pivot. While standard Quicksort can degrade to $O(n^2)$ on specific inputs, picking the middle is generally better than picking the first or last element for already-sorted data.
4.  **Recursion**: It recursively sorts the sub-slices created by the partitioning step.
5.  **Safety**: It relies on `arr.swap()` which is safe within the bounds of the slice, ensuring no undefined behavior.
