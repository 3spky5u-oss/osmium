PROMPT: Write me a quicksort implementation in Rust
RESPONSE: 

Here is a clean, idiomatic implementation of Quicksort in Rust.

This implementation is **in-place** (modifies the input slice), uses **generics** (works with any type implementing `Ord`), and respects Rust's **borrow checker** by splitting the slice into disjoint mutable borrows for recursion.

```rust
/// Public entry point for the sort.
/// It sorts the slice in-place.
pub fn quicksort<T: Ord>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    // 1. Select a pivot (Middle element is a good heuristic to avoid worst-case on sorted data)
    let len = arr.len();
    let pivot_index = len / 2;
    
    // Move pivot to the end to use standard Lomuto partition scheme
    arr.swap(pivot_index, len - 1);
    
    // 2. Partition the array
    let split_index = partition(arr);
    
    // 3. Recursively sort the sub-slices
    // We use split_at_mut to get two disjoint mutable borrows.
    // Rust's borrow checker allows this because the slices do not overlap.
    // Note: The pivot is at split_index, so we sort:
    // - left: 0..split_index (elements < pivot)
    // - right: split_index+1..end (elements > pivot)
    
    // split_at_mut returns (left, right). 
    // 'left' is 0..split_index. 
    // 'right' is split_index..end.
    // We want to exclude the pivot itself from the recursive calls.
    
    let (left, right) = arr.split_at_mut(split_index);
    
    // 'right' includes the pivot at index 0 of the slice.
    // We skip the pivot (index 0) by taking the slice starting at 1.
    if !right.is_empty() {
        // We need to ignore the pivot element at right[0]
        // We can just recurse on right[1..]
        // However, split_at_mut already took the ownership of the borrows.
        // We must ensure 'right' is not empty before indexing.
        // Since we split at 'split_index', 'right' starts at 'split_index'.
        // The pivot is at 'split_index'.
        // So 'right[0]' is the pivot.
        // We want to sort 'right[1..]'.
        // But we need to be careful: if 'right' has length 1, right[1..] is empty.
        // We can handle this by checking len.
        
        // Actually, a safer way without creating a new slice reference manually:
        // We just call quicksort on the sub-slices.
        // 'left' is already 0..split_index.
        // 'right' is split_index..end.
        // We want to sort 'right' excluding the first element (the pivot).
        // So we create a slice reference for the rest.
        
        // SAFETY: right[1..] is safe if right.len() >= 1.
        // If right.len() == 1, right[1..] is empty, which is fine.
        // We can just pass &mut right[1..].
        
        // But wait, we need to check if right is empty before indexing?
        // split_at_mut guarantees 'right' is the second part.
        // If split_index == len, 'right' is empty.
        // If split_index == len-1, 'right' is length 1.
        
        // To avoid 'right[1..]' panic on empty slice:
        if right.len() > 1 {
            let (pivot_slice, rest) = right.split_at_mut(1);
            // pivot_slice contains the pivot, we don't recurse on it.
            // rest contains the elements > pivot.
            quicksort(rest);
        }
        
        // Recurse on left
        if !left.is_empty() {
            quicksort(left);
        }
    }
}

/// Helper function to partition the array around the last element (which is the pivot).
/// Returns the index where the pivot settled.
fn partition<T: Ord>(arr: &mut [T]) -> usize {
    let len = arr.len();
    if len <= 1 {
        return 0;
    }

    let pivot = arr[len - 1];
    let mut store_index = 0;

    for i in 0..len - 1 {
        if arr[i] <= pivot {
            arr.swap(i, store_index);
            store_index += 1;
        }
    }

    // Place pivot in the correct position
    arr.swap(store_index, len - 1);
    
    store_index
}

fn main() {
    let mut data = vec![3, 6, 8, 10, 1, 2, 1];
    
    println!("Before: {:?}", data);
    quicksort(&mut data);
    println!("After: {:?}", data);

    // Test with strings
    let mut words = vec!["pear", "apple", "cherry"];
    quicksort(&mut words);
    println!("Words: {:?}", words);
}
```

### Key Implementation Details

1.  **In-Place Sorting**: The function takes `&mut [T]`, modifying the memory directly without allocating a new vector.
2.  **Generic**: It uses `T: Ord`, meaning it works with any type that can be compared (integers, strings, floats, etc.).
3.  **Borrow Checker Safety**:
    *   After partitioning, the array is split into two parts: elements smaller than the pivot and elements larger.
    *   `split_at_mut` creates two disjoint mutable borrows (`left` and `right`). Rust allows this because the slices do not overlap.
    *   We skip the pivot element (which sits at the boundary) to avoid infinite recursion.
4.  **Pivot Selection**: We pick the middle element and swap it to the end. This is a simple optimization to reduce the probability of hitting the $O(n^2)$ worst-case on already sorted data.
5.  **Base Case**: The recursion stops when the slice length is 0 or 1.

### Note on Production
While this is a correct algorithmic implementation, for production code, you should use the standard library's highly optimized and tested sort:
```rust
data.sort(); // or data.sort_unstable();
```
